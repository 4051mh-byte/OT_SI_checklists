Phase모듈설명 & 사용법
A OBS1
OBS2 		임상 관찰 체크리스트 — 평가의 첫 단계.
OBS1: 감각통합 기반 대·소근육(MC) + 감각반응(SR) 발달 단계별 관찰 항목 체크.
OBS2: 2~18세 주의력·기억·언어 등 8개 인지 영역 관찰.
✏️ 항목 체크 → 자동 달성률 산출 → JSON 내보내기로 저장.
B FLAG1
FLAG2 		Red Flag 판별 — Phase A 결과를 토대로 위험 신호 확인.
FLAG1: OBS1 기반 OT·SI·SLP·PT 통합 위험신호 & 의뢰 기준 요약.
FLAG2: OBS2 기반 인지 위험신호 요약 — 다학제 팀 공유용.
📋 연령별 위험신호 요약 & 다학제 의뢰 기준 참조.
C STD1
STD2 		표준화 평가 자동 채점 — 임상관찰 이후 표준화 검사가 필요한 경우 사용.
STD1: K-CSP2 원점수 입력 → 감각 처리 프로파일 자동 산출.
STD2: K-DTVP-2 원점수 입력 → 시지각 지수·백분위 자동 산출.
📤 결과를 JSON 내보내기로 저장해 Phase D에서 활용.
D RPT 종합 프로파일 리포트 — Phase A~C 데이터를 통합해 보고서 자동 생성.
📥 OBS1·OBS2 JSON 불러오기 + STD1·STD2 JSON 선택 통합.
전문가용 상세 보고서 / 보호자용 요약 보고서 두 가지 출력.
E ACE 중재 목표 & 활동 추천 — 평가 결과 기반 개별화 목표 수립.
📥 Phase A JSON 불러오기 → 영역별 약점 자동 감지.
중재 목표 자동 제안·세션 플래너·센터 교구 연계 활동 추천.
F MSC 다회기 비교 분석 — 동일 아동의 여러 회기 데이터를 비교.
📥 Phase A~C에서 내보낸 JSON 파일들을 한 번에 업로드.
회기별 변화 추이 차트·표 자동 생성 + Excel 내보내기.

SI-Cog CDA를 구성하는 시스템 명칭과 9개 모듈 코드의 영문 풀네임 및 의미를 정리한 참조 사전입니다.

1. 시스템 전체 명칭
약어풀네임의미
SI-Cog CDA Sensory Integration – Cognition
Clinical Decision Architecture		 감각통합(SI)과 인지(Cog) 영역을 포괄하는 임상 의사결정 아키텍처. 관찰→판별→표준화평가→종합→중재까지의 전체 워크플로우를 구조화한 시스템
2. Phase A · 임상 관찰 (Clinical Observation)
코드풀네임의미·풀이
CDA-OBS1 CDA – Observation 1
(SI Motor & Sensory Response)		 SI 기반 대·소근육(MC) + 감각반응(SR) 발달 단계별 임상관찰 체크리스트. 직접 관찰을 통해 운동능력과 감각처리 반응을 기록하는 첫 번째 관찰 모듈
CDA-OBS2 CDA – Observation 2
(Foundational Cognition)		 기초인지 임상관찰 체크리스트. 2~18세 아동의 6개 인지 영역을 관찰·기록하는 두 번째 관찰 모듈
3. Phase B · Red Flag 판별 (Clinical Flagging)
코드풀네임의미·풀이
CDA-FLAG1 CDA – Flag 1
(SI Multidisciplinary Red Flag & Referral Guide)		 OBS1 결과 기반 다학제 Red Flag 판별 및 의뢰 가이드. OT·SI·SLP·PT 관점에서 위험신호를 식별하고 타 직군 의뢰 기준을 제공
CDA-FLAG2 CDA – Flag 2
(Cognition Red Flag Summary)		 OBS2 결과 기반 인지 영역 Red Flag 요약. 다학제 팀 공유용으로 인지 위험신호를 요약·정리
4. Phase C · 표준화 평가 채점 (Standardized Scoring)
코드풀네임의미·풀이
CDA-STD1 CDA – Standardized 1
(K-CSP2 Auto Scorer)		 K-CSP2 자동 채점기. 보호자 설문 원점수를 입력하면 감각 처리 프로파일(감각 반응 패턴)을 자동 산출
CDA-STD2 CDA – Standardized 2
(K-DTVP-2 Auto Scorer)		 K-DTVP-2 자동 채점기. 시지각 검사 원점수를 입력하면 시지각 결함 유무·정도 및 시각-운동 통합 능력 결과를 자동 산출
5. Phase D · 종합 분석 (Integration)
코드풀네임의미·풀이
CDA-RPT CDA – Report
(SI-Cog Integrated Profile Report)		 SI-Cog 종합 프로파일 리포트. OBS1 + OBS2 (+ STD 결과)를 교차분석하여 전문가용/보호자용 종합 보고서를 자동 생성
6. Phase E · 중재 계획 (Intervention Planning)
코드풀네임의미·풀이
CDA-ACE CDA – Activity & goal reCommendation Engine 중재 목표 & 활동 추천 엔진. 평가 데이터 기반으로 개인화된 중재 목표 수립, 활동 추천, 세션 플래너 기능 제공
7. Phase F · 다회기 비교 분석 (Multi-Session Comparative Analysis)
코드풀네임의미·풀이
CDA-MSC CDA – Multi-Session Comparison
(Multi-Session Comparative Analysis)		 다회기 비교분석 모듈. 동일 아동의 여러 회기(시점) 평가 데이터를 비교하여 변화 추이·진전도를 분석
